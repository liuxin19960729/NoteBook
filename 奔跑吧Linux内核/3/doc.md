# 进程管理
## 进程的诞生
```
进程包含
    1.执行代码端
    2.进程的相关资源
        2.1打开的文件
        2.2挂起的信号量
        2.3内存管理
        2.4处理器状态
        2.5一个或者多个执行的线程
        2.7数据段


linux 把进程叫做任务(task)，进程控制块在linux 内核中被命名为 struct task_struct


线程(被称为轻量级进程),它是操作系统最小的调度单位
   linux内部并没有对线程有特别的调度算法或定义特别的数据结构来标识进程(使用相同的进程PCB数据结构--struct task_struct)

    内核使用clone来创建线程,clone 和 fork 类似,
        note:
          内核clone 的时候会指定哪些资源是共享那些资源是线程独立
```
